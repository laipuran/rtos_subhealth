## RFC 001: AprilTag 识别模块协议

**状态：** 草案

**修订日期：** 2026-03-29

**摘要：** 本协议定义了 AprilTag 识别模块与机器人控制模块（TonyPi/Go2）之间的数据交换格式，旨在实现跨平台的视觉引导行走。

---

### 1. 传输协议
* **格式：** JSON
* **传输方式：** MQTT / ROS Topic / UDP (由各模块实现层决定)
* **建议频率：** $\ge 10$ Hz

---

### 2. JSON 数据结构示例

```json
{
  "timestamp": 1711674061,
  "detections": [
    {
      "id": 0,
      "distance": 1250.5,
      "center_offset_x": -0.25,
      "center_offset_y": 0.12,
      "pose": {
        "roll": 20,
        "yaw": 5.2,
        "pitch": 0.0
      },
      "hamming": 0
    }
  ]
}
```

---

### 3. 参数意义说明

#### 3.1 基础元数据
* **`timestamp`**: Unix 时间戳（单位：秒）。用于判定数据实时性，防止控制补偿过时的视觉信号。
* **`detections`**: 列表格式。若画面内无目标，返回空列表 `[]`。

#### 3.2 目标定位数据
* **`id`**: AprilTag 的唯一编号。对应医疗点位（如 0: 药房, 1: 病床）。
* **`distance`**: 机器人摄像头中心到 Tag 中心点的直线距离。单位：**mm**。
* **`center_offset_x`**: 目标中心在图像水平方向的偏差。取值范围 **$[-1.0, 1.0]$**。
    * `-1.0`: 位于画面最左侧；`1.0`: 位于最右侧；`0`: 位于水平正中心。
* **`center_offset_y`**: 目标中心在图像垂直方向的偏差。取值范围 **$[-1.0, 1.0]$**。
    * `-1.0`: 位于画面顶端；`1.0`: 位于底端。用于控制头部舵机俯仰。

#### 3.3 姿态数据 (`pose`)
* **`roll`**: 翻滚角（单位：度）。Tag 绕 Z 轴的旋转，用于判定标签张贴是否倾斜或机器人是否摔倒。
* **`yaw`**: 偏航角（单位：度）。机器人正前方与 Tag 法线的夹角。用于 Go2 实现平滑侧移对准。
* **`pitch`**: 俯仰角（单位：度）。Tag 绕 X 轴的翻转程度。

#### 3.4 置信度指标
* **`hamming`**: 汉明距离（纠错比特数）。
    * `0`: 完美匹配；值越大则误报风险越高。由控制端根据此值决定是否信任该帧。

---

### 4. 实施要求
1.  **坐标归一化**：`center_offset` 必须基于图像分辨率完成归一化，以屏蔽不同摄像头配置的影响。
2.  **空集处理**：未识别到目标时必须发送空列表，不可停止发送，以便控制端判断“丢失目标”状态。
