## AprilTag 技术选型论证

### 1. 系统定位与作用
* **空间锚点 (Spatial Anchor)**：为机器人提供绝对坐标参照，消除惯性导航的累计误差。
* **位姿解算 (6-DOF Pose)**：利用四角特征点解算相对于目标的距离（$Z$ 轴）、水平/垂直偏差（$X, Y$ 轴）及三轴旋转角（**Roll/翻滚、Pitch/俯仰、Yaw/偏航**）。
* **状态触发器 (State Trigger)**：通过特定 ID 识别，驱动机器人自动切换“药房取药”、“病床问诊”或“自主充电”等高层逻辑。

### 2. 技术方案横向对比

| 维度 | **AprilTag (本项目)** | **二维码 (QR Code)** | **电子标签 (如 AirTag/UWB)** |
| :--- | :--- | :--- | :--- |
| **定位精度** | **极高**：可达厘米级与度级 | **中**：对形变敏感，远距离解算差 | **高/中**：视距传播下效果较好 |
| **硬件成本** | **极低**：仅需普通打印纸 | **极低**：仅需普通打印纸 | **高**：需额外部署基站或芯片 |
| **感知维度** | 完整 **6 自由度位姿** | 仅坐标，旋转角不精确 | 仅距离或粗略方位 |
| **鲁棒性** | **强**：低分辨率、大遮挡下可识别 | **弱**：遮挡或模糊即失效 | **强**：无视光照和遮挡 |
| **实时性** | 取决于视觉帧率（$\ge 10$ Hz） | 取决于解码效率 | 极高 |

### 3. 核心优势分析
* **低算力开销**：算法极其轻量，适合 **TonyPi** 等嵌入式平台。
* **环境自适应**：具备 **汉明距离 (Hamming Distance)** 纠错机制，在医疗环境复杂光照或机器人步态震动（如 **Go2**）时误报率极低。
* **无源部署**：无需为点位供电，仅需张贴打印标签，维护成本几乎为零。

### 4. 局限与补偿
* **局限**：对光照强度有要求，极端暗光下识别率下降。
* **补偿**：结合机器人头部舵机进行主动扫描，或在必要时开启辅助补光灯。
